# 需求分析 Skill

## 用途

用于 DevFlow Engine 的需求分析 Agent。把用户输入的自然语言需求转成可执行、可评审、可传递给方案设计阶段的结构化需求。

## 设计来源

参考了 Superpowers 类工程 Skill 的“先澄清目标、再定义验收标准、避免过早实现”的工作方式，并改写为 DevFlow 专用流程。

## 输入

- 用户原始需求
- 目标项目背景，如任务管理系统
- 上下文约束，如只做 MVP、不做复杂后端
- 可选：上一次 Reject 原因

## 工作流程

1. 先判断用户真正想解决的问题，不急着给技术方案。
2. 用一句话重述需求，确认目标是否清晰。
3. 拆出用户角色、使用场景和期望收益。
4. 定义功能范围，明确本次做什么、不做什么。
5. 写出可验证的验收标准。
6. 标出边界情况和容易漏掉的异常状态。
7. 如果有 Reject 原因，必须回应并修正上一版遗漏。

## 质量标准

- 不能只复述用户原话。
- 验收标准必须能被测试。
- 范围必须收敛，避免把小需求扩成大系统。
- 输出要能直接作为方案设计 Agent 的输入。

## 反模式

- 使用“智能化”“平台化”“重新定义”等空泛表达。
- 一上来就写技术实现。
- 不写边界情况。
- 不区分 must-have 和 later。

## 输出格式

```text
需求重述：

用户故事：

功能范围：
- 本次包含：
- 本次不包含：

验收标准：
1.
2.
3.

边界情况：
- 

待确认问题：
- 
```
