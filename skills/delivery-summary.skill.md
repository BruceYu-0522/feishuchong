# 交付总结 Skill

## 用途

用于 DevFlow Engine 的交付总结 Agent。整合全部阶段产物，生成最终交付包和 MR / PR 描述草稿。

## 设计来源

参考了工程交付 Skill 中“证据优先、列清楚变更、测试和残余风险”的方法，并改写为 DevFlow 的最终交付格式。

## 输入

- 需求分析产物
- 技术方案
- 代码变更说明
- 测试报告
- 代码评审报告
- 审批记录

## 工作流程

1. 汇总本次需求完成了什么。
2. 汇总代码变更范围。
3. 汇总测试结论。
4. 汇总代码评审结论。
5. 标出未完成事项和后续建议。
6. 生成可复制的 MR / PR 描述草稿。

## 质量标准

- 不夸大完成度。
- 明确区分已完成和后续建议。
- 测试摘要必须引用测试阶段产物。
- MR 描述要适合直接复制。

## 反模式

- 只写“本次功能已完成”。
- 不写测试情况。
- 不写风险和后续事项。
- MR 描述太空泛。

## 输出格式

```text
交付总结：

完成内容：
- 

变更摘要：
- 

测试摘要：
- 

评审结论：

残余风险 / 后续建议：
- 

MR / PR 描述草稿：
标题：
正文：
```
